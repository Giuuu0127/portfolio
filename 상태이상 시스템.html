1. 개요

PC/ NPC(Pawn)에게 "상태 이상"를 적용할 수 있는 신규 기능을 추가한다.

  • 인 게임 핵심 경험 중 "생존 / 파밍을 위한 PVE 전투"를 위한 장치이다.
  • 핵심 경험을 수행 시킬 때 "상태 이상"은 다양한 제약, 기믹으로 동작하는 장치로 만드는 것을 목표로 한다.
  • 다양한 인게임 장치에서 상태 이상를 원하는 대상에게 적용될 수 있게 구조화 한다.
    • 인게임 장치 - 날씨 / 소모성 아이템 / 탄환 / 화살 / 무기 등...
  • 상태 이상 시스템을 통해 다음과 같은 기능으로 워킹시키는 것을 목표로 한다.

1.1. 아이템 사용 시 상태이상 부여

  • 특정 아이템을 PC가 사용 시 특정 스테이터스를 [증가 / 감소] 시킨다.

1.2. 날씨에 따른 상태 이상 부여

  • 이후 마일스톤에선 시스템을 확장, 다양한 인게임 장치에서 사용할 수 있게 만든다.
    • 특정 무기를 사용해 대상에게 피해를 입힐 시 상태 이상 부여.
    • 특정 탄환 / 화살 등을 사용해 대상에게 적중 시 상태 이상 부여.
  • 모닥불 주변에 있을 경우 추위 상태의 지속 시간의 감소 속도가 가속.

1.3. 개발 목표

  • PC / NPC의 모든 파라미터를 조정(증가 / 감소)할 수 있게 만든다.
  • 상태 이상이 적용되었음을 사용자에게 알릴 수 있는 HUD UI를 고려한다.
    • 어떤 상태 이상인지 / 유효시간은 얼마나 남았는지 등...의 정보를 노출시킨다.
  • 인게임 기능에 상태 이상을 부여할 수 있도록 데이터 구조를 고려한다.
  • 상태 이상 기능을 "만능 장치"로 만들지 않는다.
    • PC에게 특정 모션을 수행 시키는 기능.
    • NPC AI 변수로 사용.
    • 조건부 변수로 사용.
    • 한 시스템이 너무 많은 장치로 활용되면 많은 제약사항이 생길 수 있다.
      • 이 점을 고려, 절대 만능 장치로 만들어내지 않는다.
  • 상태 이상을 다양한 변수들로 사용하는 것을 지양하며, 복잡한 구조로 인해 테이블 작업자가 이해하기 어렵게 만들지 않는다.

2. 상태 이상이란?

특정 인게임 장치를 트리거로 Pawn에게 [특수한 효과]를 적용하는 역할의 기능이다.

  • ex 1 - 아이템을 사용 했을 때 Pawn의 체력 N 회복.
  • ex 2 - 특정 기상 상태에 오래 위치 되었을 때 Pawn에게 디버프 적용.

다음과 같은 효과를 상태 이상 시스템을 통해 Pawn에게 부여할 수 있다.

  • Pawn의 특정 파라미터를 [증가 / 감소] 시킨다.
  • Pawn에게 특수한 효과를 주기적으로 [발생] 시킨다.
  • 모든 상태 이상은 [지속 시간]이 존재하며, 시간이 만료될 경우 Pawn에게 제거된다.
  • 상태 이상은 복수 적용이 가능하다.
    • 단, [동일한 ID]의 상태 이상은 복수 적용은 허용하지 않는다.
    • 동일한 ID가 추가 적용 시 지속 시간이 갱신된다.

3. 라이프 사이클

모든 상태 이상은 첨부된 그림과 같은 "라이프 사이클"이 존재한다.

  • 라이프 사이클엔 "총 4개"의 상태가 존재하며, 각 상태별 동작은 다음과 같다.

3.1. Attach

Pawn에게 상태 이상이 "최초 적용"되는 상태이다.

  • Attach된 순간부터 상태 이상에 정의된 "효과 / Duration"이 적용된다.
  • Attach를 시키기 위해선 다음과 같은 인게임 트리거 장치가 필요하다.
    • 날씨 시스템
    • 아이템 사용 시스템
  • Pawn에게 이미 동일한 ID의 "상태 이상"이 적용된 상태라면 "Duration"이 갱신된다.
    • 동일한 ID의 상태 이상은 "중첩"을 허용하지 않는다.

3.2. Duration

Pawn에게 상태 이상이 Attach된 이후 "유지되는 시간"이다.

  • 해당 시간만큼 Pawn에게 유지되며, 시간이 만료될 경우 Detach가 된다.
  • 유지시간은 상태 이상 별로 작성이 가능하며, 이는 [* 데이터]를 통해 정의한다.

3.3. Period

Pawn에게 "주기적"으로 특정 효과를 발생 시킬 수 있는 상태이다.

  • 주기는 상태 이상 별로 적용 가능하며, 이는 [* 데이터]로 정의한다.

3.4. Detach

Pawn에게서 상태이상이 "제거"되는 상태이다.

  • 정의된 Duration 시간이 만료될 경우 적용된다.

4. HUD상 표현

사용자에게 적용된 상태 이상은 첨부된 그림과 같이 HUD 상에 아이콘으로 출력한다.

  • 나로 인해 대상(PvP 대상 PC / NPC)에게 [상태 이상]을 적용 시킬 경우의 표현.
    • 내게 적용된 상태 이상을 표현하고 이를 설명할 수 있는 UI 장치.
  • 상태 이상의 HUD 표현은 두가지로 구분해 출력하며, 내용은 다음과 같다.

4.1. 상태 이상 Attach 연출

첨부된 그림과 같이 "PC 주변"에 Attach된 상태이상의 아이콘이 노출된다.

  • Attach되는 순간 동시 출력되며, 연출 시간은 모두 동일하다.
  • 상태이상의 아이콘은 모두 [* 데이터]를 통해 등록할 수 있게 구조화한다.

<예외 처리 - 동일 시점 복수 적용 시>

동일 시점에 복수의 상태 이상이 적용될 경우 Attach 연출은 "1종"만 출력된다.

  • 나머지 효과들은 무시한다.
  • 예를 들어 A / B / C 3종의 상태 이상이 동시 적용될 경우, A의 Attach 연출만 출력되고 나머지 [B / C]에 대한 연출은 무시한다.
  • 너무 많은 Attach 연출로 인해 HUD가 복잡해지는 것을 방지하기 위한 처리이다.

4.2. 적용된 상태이상 표현

HUD UI 상 "상태이상"이 그림과 같이 노출될 때 다음과 같은 사항들이 가시적으로 표현되어 사용자에게 정보를 제공한다.

  • 지속시간.
  • 효과를 가시화 시킨 아이콘.
  • "이로운 / 해로운 효과"에 대한 아이콘 컬러 정보.

5. StatusEffectTable

"상태이상"과 관련된 내용을 데이터로 정의할 수 있도록 [StatusEffectTable.xlsx] 파일을 추가한다.

  • 엑셀 파일 내에 [StatusEffectInfo Sheet]를 추가, 다음과 같은 정보를 구성할 수 있게 구조화한다.

StatusEffectTable.xlsx

5.1. id

테이블 데이터를 고유한 숫자 값으로 구성된 ID를 지정하는 역할의 컬럼이다.

  • 데이터 형식은 int이다.
  • 값이 없거나, [StatusEffectInfo]에 중복 값을 허용하지 않는다.

5.2. code_name

테이블 데이터를 고유한 문자열로 구성된 Codename로 지정하는 역할의 컬럼이다.

  • 데이터 형식은 string이다.
  • 값이 없거나, [StatusEffectInfo]에 중복 값을 허용하지 않는다.

5.3. icon

상태 이상이 적용되었을 경우 어떤 아이콘으로 인게임 상에 출력할지 결정하는 역할의 컬럼이다.

  • 데이터 형식은 string이며, 아이콘 리소스의 경로를 작성한다.
  • 해당 값을 통해 [* HUD UI]에 적용된 상태이상을 아이콘으로 표현한다.

5.4. duration

상태이상의 지속시간을 결정하는 역할의 컬럼이다.

  • 데이터 형식은 int이며, 단위는 sec이다.
  • 값이 0일 경우 무제한 지속되는 상태이상으로 판단한다.

5.5. period

상태 이상 효과의 [발동 주기]를 결정하는 역할의 컬럼이다.

  • 데이터 형식은 float이며, 단위는 sec이다.
  • 값이 0일 경우 발동 주기가 존재하지 않는 상태이상으로 판단한다.

5.6. change_parameter

상태이상이 지속되는 동안 Pawn의 특정 파라미터를 조정하는 역할의 컬럼이다.

  • 데이터 형식은 구조체이며, 복수 입력이 가능하다.
  • 구조체의 구성 및 인자별 역할은 다음과 같다.
{(ParameterType, ChangeType, Value)}

5.6.1. ParameterType

상태 이상 적용시 Pawn의 어떤 파라미터를 조정할지 결정하는 역할의 인자다.

  • 데이터 형식은 enum이며, 신규 enum타입인 [ParameterType]을 사용한다.
  • [ParameterType]의 구성은 다음 표와 같으며, ParameterType에 정의되지 않은 값은 허용하지 않는다.
ParameterType 내용
max_hp Pawn의 [최대 체력]을 증가 / 감소 시켜주는 역할의 파라미터이다.
max_stamina Pawn의 [최대 스테미너]를 증가 / 감소 시켜주는 역할의 파라미터이다.
max_weight Pawn의 [최대 인벤토리 중량]을 증가 / 감소 시켜주는 역할의 파라미터이다.
base_speed Pawn의 [기본 이동 속도]을 증가 / 감소 시켜주는 역할의 파라미터이다.
🎯 Pawn에게 다양한 파라미터들이 생기게 될 경우 관련된 Type을 추가, 파라미터를 조정할 수 있게 구조화한다.
❌ ParameterType에 정의되지 않은 내용은 작성할 수 없다.

5.6.2. ChangeType

파라미터를 [어떻게] 조정할지 결정하는 역할의 인자다.

  • 데이터 형식은 enum이며, 신규 enum 타입인 [ChangeType]을 사용한다.
  • [ChangeType]의 구성은 다음과 같다.
ChangeType 설명
inc 절대 값 증가
dec 절대 값 감소
percent_inc 비율 증가
percent_dec 비율 감소

5.6.3. Value

지정한 파라미터를 얼마나 [증가 / 감소] 시킬지 값을 결정하는 역할의 인자다.

  • 데이터 형식은 float이다.

5.6.4. 데이터 동작 예시

{(max_hp, inc, 500)}

change_parameter 컬럼 값이 위와 같이 작성될 경우를 가정 하에 데이터 동작의 예시를 설명한다.

  • 해당 상태이상이 적용된 Pawn은 "최대 체력이 500 증가"한다.
  • 상태이상의 지속시간이 만료되면 최대 체력이 본래 값으로 돌아간다.

{(max_hp, percent_inc, 0.5)}

  • 해당 상태이상이 적용된 Pawn의 현재 체력에서 50%가 추가 증가된다.
  • pawn의 체력이 10,000인 것을 가정 하에 예시는 다음과 같다.
    • 10,000의 50%인 5,000이 증가한다.
  • 상태이상의 지속시간이 만료되면 최대 체력이 본래 값으로 돌아간다.

5.6.5. 복수 적용 예시

첨부된 그림과 같이 각기 다른 상태 이상이 동일한 [ParameterType]으로 Pawn에게 적용될 경우를 가정해 예시를 설명한다.

  • ParameterType별로 일괄 계산해, Pawn의 상태이상 결과 수치 값을 도출한다.
계산식 = (기본 값 + inc - dec) * ((percent_inc / 100) - (percent_dec / 100) + 1)
  • Pawn에게 적용된 상태이상 1종이 만료될 경우 계산이 다시 이루어지며, 결과 값에 대한 업데이트가 즉시 이루어질 수 있게 만든다.

5.7. period_effect

상태 이상의 [* 주기]마다 Pawn에게 어떤 효과를 발동할지 결정하는 역할의 컬럼이다.

  • 데이터 형식은 구조체이며, 복수 입력이 가능하다.
  • 구조체의 구성 및 인자별 역할은 다음과 같다.
{(PeriodEffectType, ChangeType, Value)}

5.7.1. PeriodEffectType

주기가 발동될 때 "어떤 효과"를 적용할지 결정하는 역할의 인자다.

  • 데이터 형식은 enum이며, 신규 enum타입인 "PeriodEffectType"을 사용한다.
  • PeriodEffectType의 구성은 아래 표와 같다.
PeriodEffectType 설명
hp Pawn의 [현재 hp]를 주기적으로 증가 / 감소 시켜주는 역할의 Type이다
stamina PC의 [현재 stamina]를 주기적으로 증가 / 감소 시켜주는 역할의 Type이다.
life_time_modifier PC의 [현재 남은 lifetime]을 주기적으로 증가 / 감소 시켜주는 역할의 Type이다.
🎯 주기적으로 변경되어야 할 Pawn의 수치가 추가된다면 해당 Type을 추가해 적용할 수 있게 구조화한다.
❌ PeriodEffectType에 정의되지 않은 값은 허용하지 않는다.

5.7.2. ChangeType

PeriodEffectType을 [어떻게] 조정할지 결정하는 역할의 인자다.

  • 데이터 형식은 enum이며, 신규 enum 타입인 [ChangeType]을 사용한다.
  • ChangeType의 구성은 다음과 같다.
ChangeType 설명
inc 절대 값 증가
dec 절대 값 감소
percent_inc 비율 증가
percent_dec 비율 감소

5.7.3. Value

PeriodEffectType을 얼마나 증가 / 감소 시킬지 "값"을 결정하는 역할의 인자다.

  • 데이터 형식은 float이다.

5.7.4. 데이터 적용 예시

{(hp, dec, 50)}

PeriodEffect가 위와 같이 작성되어 있고 "LifeTime이 1분 / Period가 2초"로 적용될 경우 해당 데이터는 인게임 상에서 다음과 동작한다.

  • 동작 예시 - 1분동안 Pawn의 hp가 2초당 50씩 감소한다.

{(hp, inc, 50)}

"LifeTime이 1분 / Period가 1초"일 경우

  • 동작 예시 - 1분동안 Pawn의 hp가 1초당 50씩 회복한다.

{(stemina, percent_inc, 0.2)}

"LifeTime이 1분 / Period가 10초"일 경우.

  • 동작 예시 - 1분동안 Pawn의 stemina가 10초당 20%씩 회복한다.

{(stemina, percent_dec, 0.1)}

"LifeTime이 1분 / Period가 10초"일 경우.

  • 동작 예시 - 1분동안 Pawn의 stemina가 10초당 10%씩 감소한다.

예외 처리

"percent_inc / percent_dec"계산 시 결과가 소수이면 소수점 이하는 버림처리한다.

  • PeriodEffect로 인해 HP가 100.9가 증가될 경우 버림 처리되어 100이 증가된다.

5.7.5. 복수 적용 예시

첨부된 그림과 같이 "상태 이상 A ~ C"가 동시 적용될 경우를 가정 하에 예시를 설명한다.

  • 주기 효과는 상태 이상 별로 계산되며, 주기 효과가 적용되는 기대 결과는 첨부된 타임라인 그림과 같다.

5.8. detach_groggy

PC가 그로기 상태가 되었을 때 적용된 상태 이상의 해제 여부를 결정하는 역할의 컬럼이다.

  • 데이터 형식은 boolean이며, 각 값에 따른 동작은 다음과 같다.
    • TRUE - PC가 그로기 상태가 될 경우 남은 Duration과 관계 없이 해제된다.
    • FALSE - PC가 그로기 상태가 되어도 유지된다.
